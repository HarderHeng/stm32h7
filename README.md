# Atom01 Robot Firmware (Rust on STM32H7)

Standalone Rust port of the ROBOTO_ORIGIN humanoid robot control pipeline — inference + actuation running entirely on STM32H743, no Linux, no ROS, no ONNX Runtime.

## Status

| Phase | Description | Status |
|---|---|---|
| 0 | Embassy scaffold + USB-CDC shell | ✅ Done (48 KB ELF) |
| 1 | INT8 quantization tooling (Python) | ✅ Done (synthetic data; pass through `--strict` with real trained `policy.onnx`) |
| 2 | MLP forward pass (INT8 GEMM, 4 layers) | ✅ Done (real weight binaries embedded) |

| 4 | DAMIAO motor MIT frame codec | ✅ Done (`encode_mit`, `decode_mit_feedback`) |

| 10 | README + deployment scripts | ✅ This file |

## Build

```bash
# Install target
rustup target add thumbv7em-none-eabihf

# Build release firmware
cargo build --release

# Output: target/thumbv7em-none-eabihf/release/stm32h7 (~48 KB)
```

## Flash

```bash
# Via ST-Link V2 (probe-rs auto-detects chip from .cargo/config.toml)
cargo run --release

# Or explicitly
probe-rs download --chip STM32H743ZI target/thumbv7em-none-eabihf/release/stm32h7
probe-rs reset    --chip STM32H743ZI
```

## View Logs (RTT)

```bash
probe-rs rtt --chip STM32H743ZI
```

The firmware brings up a USB-CDC shell at the same time as the control loops. Connect a USB cable to the OTG_FS port and use any serial monitor (115200 baud) to interact.

## Quantize a Trained Policy

```bash
# Install Python deps (one-time)
pip install onnx onnxruntime numpy

# Quantize upstream policy.onnx → INT8 weights
python3 tools/quantize_onnx.py \
    --input  /path/to/policy.onnx \
    --output weights/ \
    --strict    # fails if max relative error > 1%

# Outputs:
#   weights/w{1..4}_int8.bin   - raw i8 bytes, row-major [out_dim × in_dim]
#   weights/b{1..4}_int8.bin   - raw i8 bytes
#   weights/manifest.json      - shapes, scales, validation result
#   weights/test_fixtures.json - 10 random obs + FP32 reference outputs (for Rust tests)
```

The Rust firmware embeds the four weight binaries and the four bias binaries via `include_bytes!`. After running the quantizer, rebuild the firmware and reflash.

## Algorithm Pipeline

```
[IMU 500Hz]                  [23 motor encoders 250Hz]
      │                              │
      └─────────┬────────────────────┘
                ↓
       ┌─────────────────────┐
       │  observation push   │ 78 floats → ring buffer (10 frames)
       └────────┬────────────┘
                ↓
       ┌─────────────────────┐
       │   MLP inference     │ 780 floats → 23 floats (INT8 GEMM, 4 dense + ELU)
       │   (every 20 ms)     │
       └────────┬────────────┘
                ↓
       ┌─────────────────────┐
       │  action post-proc   │ clip → scale 0.25 → USD→URDF → +default_angle
       └────────┬────────────┘
                ↓
       ┌─────────────────────┐
       │  ankle decoupling   │ (4 motors) FK → PD in joint space → J^T → motor τ
       │                     │ (19 motors) motor pos target × motor_sign
       └────────┬────────────┘
                ↓
       ┌─────────────────────┐
       │  MIT frame build    │ 23 × 8-byte CAN frames
       │  (every 4 ms)       │
       └────────┬────────────┘
                ↓
       ┌─────────────────────┐
       │  CAN transmit       │ 4 buses × 1 Mbps
       └─────────────────────┘
```

## Module Reference

| Module | What it does |
|---|---|
| `mlp::Mlp::forward_int8` | 4-layer INT8 quantized MLP with ELU; weights embedded via `include_bytes!` |
| `ankle::AnkleDecoupler` | IK / FK / Jacobian for the closed-chain ankle (4-bar parallel mechanism); Newton iteration for FK |
| `canproto::encode_mit` / `decode_mit_feedback` | DAMIAO motor 8-byte MIT frame; `range_map` with round-half-away-from-zero |
| `hal::CanBus` / `SerialPort` / `SpiFlash` | Hardware abstraction traits; mock implementations for host tests |
| `observation::Observation` | 10-frame ring buffer + flatten to 780-float MLP input |
| `drivers` | Embassy driver stubs (Phase 5–9 work) |

## Known Limitations & Next Steps

1. **Drivers are stubs.** `src/drivers.rs` has the trait-impl skeleton but bodies return errors (CanBusHw returns `Err(CanError::Hardware)` to avoid silent frame loss). Wire to `embassy_stm32::can` (bxCAN) and `embassy_stm32::usart` (USART4) for real hardware.
2. **Only `inference_task` is spawned.** `bin/control.rs` runs the 50 Hz MLP inference loop but the 250 Hz `control_task` that calls `step_control()` requires real Embassy CAN/IMU wiring and is deferred to Phase 6/7. `cmd_vel` is currently hardcoded to `[0.0; 3]` (robot holds home pose) until a real source is wired (USB shell, ROS2 topic, wired remote).
3. **Tests run on target only.** Embassy deps don't build for host x86_64. Use `probe-rs test --chip STM32H743ZI` to execute unit tests on hardware.
4. **Synthetic weights in `weights/` were generated by the quantizer's fallback** (since real `policy.onnx` wasn't available). The build script (build.rs) prints a loud `WARNING` when `manifest.json` reports `validation_passed=false`. Set `STRICT_WEIGHTS=1` to make it a hard panic in CI/release. To use real weights: `python3 tools/quantize_onnx.py --input <policy.onnx> --output weights/ --strict`.
5. **STM32H743ZI vs VIT.** `Cargo.toml` enables the `stm32h743zi` feature (176-pin). For 100-pin TFBGA (VIT), change to `stm32h743v` and re-pin peripherals.
